本工程按“决策链生成Agent→节点调度Agent→单元任务执行Agent→决策融合”实现最小可运行科研骨架：决策链生成输出任务有向图（节点含输入/输出/handler），节点调度按依赖触发执行器，执行器调用 mock 数据与简化算法产出结果，融合模块当前采用单模型选择策略并把结果写入共享数据池。数据层以 pandas DataFrame 的 MockDataGenerator 生成降雨/来水并提供一致性校验。日志采用 loguru 按日期落盘。

与 架构.md 的差异：真实数据库/小模型/规则引擎/水动力模型尚未接入；多模型融合与链路优化插件未实现，先用可替换接口与默认单模型策略占位。
